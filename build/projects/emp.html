EMP: Environmental Music Player

Motivation

Our inspiration for this project was the popular video game Animal Crossing: New Horizons, which plays different musical tracks for different times of day and weather conditions. We thought that that kind of musical functionality might be nice to have in real life.

Think of you’re stuck indoors working on homework all day, it can be easy to forget about the external world. With dynamically adaptive music, we hope to help users maintain a feeling of connection with the great outdoors.

Project Description

Based on a connected Raspberry Pi’s location, this project adaptively plays different soundtracks depending on the location’s current precipitation, sunrise/sunset times, lighting conditions, and/or temperature. The music plays in a web frontend, through which the user can also specify what environmental factors should play a role in the tracks played.

Our system has two base tracks, one for “daytime” and one for “nighttime.” The base tracks are chosen based on time or light level. Each track has three variations: one for clear weather, one for rainy weather, and one for snowy weather. The variations are chosen based on local weather conditions or temperature.

Approaches

First, our Web UI requests weather and daylight information from a Raspberry Pi with a specified ID. Because the requests are made via MQTT, the user can select any Raspberry Pi at any location. The Raspberry Pi then uses a geolocation API to approximate its location from its IP address. Next, the Pi requests information about the weather and daylight for its specific location from a weather API. Finally, the Pi gets a light reading and temperature reading from its local sensors, before bundling all of that information into a payload that gets sent to the Web UI.
